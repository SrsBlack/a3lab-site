How OpenClaw fits our stack

Think of our stack like a person. Perplexity is the researcher — it finds real-time information from the web. Claude is the thinker — it reasons, writes, and builds logic. OpenClaw is the doer — the hands and legs that click buttons, send messages, organize files, and run daily routines across your apps. Zapier and Make are the plumbing that connects everything to your existing tools.

Perplexity The research brain. Finds real-time information with sources.
Claude The reasoning brain. Writes, thinks, drafts, builds logic.
OpenClaw The hands and legs. Clicks, types, sends, files, runs routines 24/7.
Zapier / Make The plumbing. Connects CRM, Notion, Google Workspace, and 5,000+ tools.

Under the hood, we deploy OpenClaw as a containerized Gateway service — typically Docker on a VPS (DigitalOcean, Hetzner, or Contabo) with Nginx reverse proxy, SSL via Let's Encrypt, and systemd for process management. Model routing goes through OpenRouter or direct provider APIs (Anthropic, OpenAI, Google), and we configure per-agent model overrides so cost-sensitive tasks use lighter models while complex reasoning hits Claude or GPT-class models. Multi-channel routing is handled at the Gateway layer, with session isolation per channel and persistent memory stored as plaintext Markdown files you can inspect and edit. We configure MCP integrations for browser automation, file operations, and external service connections, and set up cron jobs for scheduled tasks with proper retry policies and session management. Skills are vetted before installation, and we harden every deployment with non-root users, firewall rules, and scoped API permissions. Everything is documented, reproducible, and yours to own.
